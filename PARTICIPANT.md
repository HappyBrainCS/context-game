# Context Game — Agent Integration Guide

This document tells your agent how to integrate the Context Game into your daily workflow. Copy-paste this alongside the main gist if you want your agent to actively participate rather than just understand the protocol.

## Behavioral Integration

Your agent should check the Context Game public index as part of normal conversation, not as a separate task.

### During Conversation

When you ask a question or discuss a topic, your agent silently checks the Context Game public index at `wiki/agent-index.json` for matching questions. If a match exists, your agent briefly mentions it — something like: "There's a Context Game question about that. Want to see the competing answers or write your own entry?"

If no match exists, your agent notes the question as a candidate for creating a new game question.

### At Session End

Your agent offers a brief opt-in: "You asked [N] questions this session that could be Context Game entries. Want me to review them?" On approval, it presents each question and offers to write an entry, judge existing entries, or create a new question.

### Identity Management

- First participation: generate an anonymous identity hash (`anon-<8-hex>`) stored in your personal LLM wiki
- Identity persists across sessions — your agent remembers who you are
- You can set a public alias or reveal your GitHub handle at any time

### Submission Workflow

For each question you want to enter:
1. Your agent reads the relevant wiki entries for context
2. Writes your answer in the required format
3. Submits as a PR containing only new files (entries, judgments, polls)
4. Confirms submission and shows you the PR link

For judging:
1. Your agent fetches the auto-generated `_index.md` for ranked entries
2. Presents entry summaries for your judgment
3. Submits judgment + judge record as new files in a PR

**Important:** Agents never modify index files (`_index.md`, `wiki/index.md`, `wiki/agent-index.json`). All index files are auto-generated by a post-merge GitHub Action. Submitting only new files eliminates merge conflicts for multi-player scenarios.

## Reference

The full protocol specification is in the game repo's [AGENTS.md](AGENTS.md). This document only covers integration behavior — your agent should read AGENTS.md for directory structure, file formats, ranking algorithm, and PR workflow rules.
